\begin{posterbox}[name=abstract,column=0,row=0,height=0.175]{Abstract}
Monocular relative pose estimation supports autonomous \textbf{Rendezvous, Proximity Operations, and Docking (RPOD)} and \textbf{On-Orbit Servicing, Assembly, and Manufacturing (OSAM)} missions. These operations may involve noncooperative client spacecraft that do not actively or passively communicate state information to the chaser spacecraft. We demonstrate a semi-unseen model-based pose estimation pipeline that takes as input an \textbf{Red-Green-Blue (RGB)} image of the target and its \textbf{Computer-Aided Design (CAD)} model, and the direct pose regression models are generic such that the direct pose regression components do not train on client specific imagery. The system is tested onboard a robotic \textbf{Hardware-in-the-Loop (HIL)} testbed with motion-capture-based truth.
\end{posterbox}

\begin{posterbox}[name=motivation,column=0,below=abstract,height=0.165]{Motivation and Problem Setup}
\begin{itemize}\tightlist
  \item Non-cooperative clients may not transmit state information, thus, the chaser must infer pose onboard
  \item Client-specific pose networks require labeled or synthetic imagery and retraining for each known target
  \item This model-based pipeline attempts to avoid client-specific pose-training for the direct pose regression components
  \item HIL testing subjects the pipeline to the sim2lab gap, an approximate mockup, and \textbf{Size, Weight, Power, and Cost (SW\textup{a}P-C)} constraints
\end{itemize}
\end{posterbox}

\begin{posterbox}[name=platform,column=0,below=motivation,above=bottom,textfont={\compactpostertext}]{Experimental Platform and Truth Data}
\centeredgraphic[width=0.98\linewidth,height=0.375\colheight,keepaspectratio]{panel_platform.pdf}
\figcap{Two holonomic robots emulate chaser-client motion; the lower view shows the onboard sensing and compute stack.}
\subhead{Demonstration Stack}
\begin{itemize}\tightlist
  \item Camera: Basler Ace (acA4112-30uc)
  \item Compute: NVIDIA Jetson Orin NX 16 GB
  \item Emulation uses Distance Unit (DU) ratio 1:20 and Time Unit (TU) ratio 10:1
  \item Orchestrated with Robot Operating System 2 (ROS 2): nodes run the measurement model/filter, collect imagery, and raw Vicon data
\end{itemize}
\subhead{Truth Collection and Refinement for HIL Testing}
\begin{itemize}\tightlist  
  \item Vicon motion capture data is processed to compute the relative poses between the camera and the printed client, however, the truth camera and truth client frames are only approximated in Vicon so their offsets need to be calculated
  \item After Open Source Computer Vision Library (OpenCV) calibration, a nonlinear least squares optimization problem is solved to estimate offsets and recalculate the relative pose between the camera and target
  \item Residual error can arise from calibration noise, Vicon coverage limits, and print/CAD mismatch
\end{itemize}
\end{posterbox}

\begin{posterbox}[name=method,column=1,row=0,height=0.430]{Method Overview}
\centeredgraphic[width=0.99\linewidth,height=0.225\colheight,keepaspectratio]{fig_pipeline_y2_rpe.png}
\figcap{The measurement model alternates between recovery and frame-to-frame tracking; feasibility and inlier checks determine publication or reacquisition.}
\vspace{1em}
\subhead{Inputs}
\begin{itemize}\tightlist
  \item Monocular RGB image, target CAD/mesh model, camera calibration, and filter prediction
\end{itemize}
\subhead{Recovery Mode}
\begin{itemize}\tightlist
  \item Localize the client and initialize pose with model-based recovery
  \item Check detector support against the projected model and seed image-to-model correspondences
  \item The first successful recovery mode pass initializes the filter
\end{itemize}
\subhead{Tracking Mode}
\begin{itemize}\tightlist
  \item Propagate correspondences with optical flow
  \item Run \textbf{Perspective-n-Point (PnP)}, refine, monitor inliers, and fall back to recovery if tracking degrades
\end{itemize}
\end{posterbox}

\begin{posterbox}[
  name=conclusion,
  column=2,
  below=tracking,
  above=bottom,
  textfont={\compactpostertext},
  boxpadding=0.42em
]{Conclusions and Future Work}

\subhead{Takeaways}
\begin{itemize}\tightlist
  \item Demonstrated an unseen model-based monocular pose pipeline on robotic HIL hardware with onboard processing and filter integration
  \item Recovery plus tracking combines global reinitialization with optical-flow correspondence propagation and PnP/refinement
  \item The experiment directly exercises truth acquisition, sim-to-lab constraints, and edge-compute throughput
\end{itemize}

\subhead{Next Steps}
\begin{itemize}\tightlist
  \item Validate over many runs and targets
  \item Tune timing, data-rate synchronization, and filter covariance for smoother updates
  \item Improve rendering/refinement speed with template warping
  \item Test in another facility and transition modular ROS 2 nodes to other groups
\end{itemize}
