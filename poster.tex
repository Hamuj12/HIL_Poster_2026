\begin{posterbox}[name=abstract,column=0,row=0,height=0.160]{Abstract}
Monocular relative pose estimation supports autonomous \textbf{Rendezvous, Proximity Operations, and Docking (RPOD)} and \textbf{On-Orbit Servicing, Assembly, and Manufacturing (OSAM)} for non-cooperative spacecraft. We demonstrate an unseen model-based recovery and tracking pipeline that uses a monocular \textbf{Red-Green-Blue (RGB)} image and target \textbf{Computer-Aided Design (CAD)} model rather than target-specific pose-training imagery. The system runs on a robotic \textbf{Hardware-in-the-Loop (HIL)} testbed with motion-capture-based truth correction and onboard compute.
\end{posterbox}

\begin{posterbox}[name=motivation,column=0,below=abstract,height=0.172]{Motivation and Problem Setup}
\begin{itemize}\tightlist
  \item Non-cooperative targets may not transmit relative state or provide fiducials; the chaser must infer pose onboard.
  \item Target-specific pose networks require labeled or synthetic imagery and retraining for each known target.
  \item This model-based pipeline assumes a target three-dimensional (3D) model and avoids target-specific pose-training images.
  \item HIL testing exposes sim-to-lab appearance, compute limits, truth collection, and mockup fidelity.
\end{itemize}
\end{posterbox}

\begin{posterbox}[name=platform,column=0,below=motivation,above=bottom]{Experimental Platform and Truth Data}
\centeredgraphic[width=0.98\linewidth,height=0.342\colheight,keepaspectratio]{panel_platform.pdf}
\figcap{Two holonomic robots emulate chaser-target motion; the lower view shows the onboard sensing and compute stack.}
\subhead{Demonstration Stack}
\begin{itemize}\tightlist
  \item Camera: Basler Ace (acA4112-30uc); compute: NVIDIA Jetson Orin NX 16 GB.
  \item Emulation uses Distance Unit (DU) ratio 1:20 and Time Unit (TU) ratio 10:1.
  \item Robot Operating System 2 (ROS 2) nodes run the measurement model/filter and collect imagery plus Vicon data.
\end{itemize}
\subhead{Truth-Like Poses}
\begin{itemize}\tightlist
  \item Vicon motion capture supplies truth-like camera-target poses, but its frame definitions introduce offsets.
  \item Open Source Computer Vision Library (OpenCV) calibration estimates camera-to-board correction terms.
  \item Residual error can arise from calibration noise, Vicon coverage limits, and print/CAD mismatch.
\end{itemize}
\end{posterbox}

\begin{posterbox}[name=method,column=1,row=0,height=0.445]{Method Overview}
\centeredgraphic[width=0.99\linewidth,height=0.225\colheight,keepaspectratio]{fig_pipeline_y2_rpe.png}
\figcap{The measurement model alternates between recovery and frame-to-frame tracking; feasibility and inlier checks determine publication or reacquisition.}
\subhead{Inputs}
\begin{itemize}\tightlist
  \item Monocular RGB image, target CAD/mesh model, camera calibration, and filter prediction.
\end{itemize}
\subhead{Recovery Mode}
\begin{itemize}\tightlist
  \item Localize the target and initialize pose with model-based recovery.
  \item Check detector support against the projected model and seed image-to-model correspondences.
\end{itemize}
\subhead{Tracking Mode}
\begin{itemize}\tightlist
  \item Propagate correspondences with optical flow.
  \item Run \textbf{Perspective-n-Point (PnP)}, refine, monitor inliers, and fall back to recovery if tracking degrades.
\end{itemize}
\end{posterbox}

\begin{posterbox}[name=conclusion,column=2,below=quant,above=bottom]{Conclusions and Future Work}
\subhead{Takeaways}
\begin{itemize}\airylist
  \item Demonstrated an unseen model-based monocular pose pipeline on robotic HIL hardware with onboard processing and filter integration.
  \item Recovery plus tracking combines global reinitialization with optical-flow correspondence propagation and PnP/refinement.
  \item The experiment directly exercises truth acquisition, sim-to-lab constraints, and edge-compute throughput.
\end{itemize}
\subhead{Next Steps}
\begin{itemize}\airylist
  \item Validate over many runs and targets.
  \item Tune timing, data-rate synchronization, and filter covariance for smoother updates.
  \item Improve rendering/refinement speed with template warping.
  \item Test in another facility and transition modular ROS 2 nodes to other groups.
\end{itemize}
